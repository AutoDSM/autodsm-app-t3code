What Makes It The Agentic Storybook

📁
Folder-Based Browsing
Components organized by your folder structure. Buttons/, Cards/, Forms/ — just like your codebase.
🎯
Isolated Rendering
WebContentsView renders each component in isolation with full provider tree support.
🎛️
Prop Controls
Every prop gets a control. Strings, booleans, enums, colors — all auto-generated.

Token Inspector
See which BrandProfile tokens are applied. Semantic colors, spacing, radii.
💬
Natural Language
"Make it rounded with primary-600" — prompts that understand your design system.
⚡
Instant Preview
Every AI change reflects immediately. HMR keeps the preview live.
📝
Diff Review
See exactly what changed. Approve or reject per-hunk before committing.
🔀
Git-Native
Branch per session, auto-commit, PR creation — all built in.

Key Files & Responsibilities

The core modules that power AutoDSM

🖼️ Renderer (React)
ComponentSidebar.tsx Folder-based component tree with search, filtering, and selection
ComponentCanvas.tsx Preview area with WebContentsView, viewport controls, zoom
PropControlsPanel.tsx Auto-generated controls for component props based on TypeScript types
PromptBar.tsx AI prompt input with agent status, diff preview trigger
DiffReviewModal.tsx Side-by-side diff view with per-hunk approve/reject
⚙️ Main Process (Node.js)
ComponentIndexer.ts AST scanning service that discovers React components and extracts prop types
RenderRuntime.ts Manages Vite sidecar lifecycle, WebSocket connection, HMR coordination
BrandProfileService.ts Loads and manages design tokens, provides context to AI agents
AgentService.ts Orchestrates AI providers (Claude, Codex), manages ChangeSet generation
🔥 Vite Sidecar
vite.config.ts Preview server config with HMR, React plugin, Tailwind
preview-entry.tsx Renders selected component with provider tree, receives prop updates

IPC Channels Reference

Typed communication between processes

Channel Direction Payload Description
component:list Main → Renderer ComponentTree[] Indexed component hierarchy
component:select Renderer → Main { id, path } User selected a component
agent:prompt Renderer → Main { prompt, componentId } AI edit request
agent:stream Main → Renderer { chunk, done } Streaming AI response
changeset:apply Main → Renderer ChangeSet Diff ready for review
git:commit Renderer → Main { message, hunks[] } Commit approved changes

AutoDSM — The Agentic Storybook

Storybook taught us to browse components in isolation. Cursor taught us to edit code with AI. AutoDSM fuses both: browse your component library like Storybook, edit with natural language like Cursor, and see every change reflected instantly in an isolated, token-aware preview. The Agentic Storybook.
